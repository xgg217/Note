# 各个环节的细节

## 身份1 协作者流程

+ 和管理员开发基本相同，差异点：

  + 管理员需要添加协作者
  + PR时必须review


## 身份2 非协作者(陌生人)流程

+ 步骤1：前期准备

  + fork仓库

  + 克隆fork仓库

  + 添加原仓库源

    ```bash
    # 添加上游
    git remote add upstream https://github.com/原作者/仓库名.git

    # 查看远程源（确认 origin 是你的 Fork， upstream 是原仓库(名字可以随意，一般使用 upstream )）
    git remote -v
    ```

+ 步骤2：贡献流程，和之前的流程一致，区别点：

  + 区别1：拉取最新代码

    ```bash
    git switch main
    git fetch upstream
    git merge upstream/main

    # 推回自己的 Fork（推回自己的远程仓库）
    git push origin main
    ```

  + 区别2：提交PR

    + 需要到自己仓库主页提交PR
