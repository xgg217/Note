# 暂存和提交

## 查看当前状态

+ 查看当前状态

  ```bash
  git status
  ```

## 提交步骤

1. 暂存
2. 将暂存的内容一起提交

### 暂存

+ 方式1 暂存指定文件 `git add <file1> <file2>`

  ```bash
  git add a.js
  ```

+ 方式2 暂存当前目录及子目录下的所有文件

  ```bash
  git add .
  ```

### 将暂存的内容一起提交

+ 方式1 `git commit` 会打开默认文本编辑器，你可以在编辑器中编写提交信息

  + 设置默认编辑器 `git config --global core.editor "code --wait"`

+ 方式2 `git commit -m "提交信息"` 直接在命令行中编写提交信息，不会打开文本编辑器

## 提交消息修改

+ 提交消息修改

  ```
  git commit --amend
  ```

+ 该命令通常用于修改最新的提交消息

+ 所以使用该命令时，要确保自己所在的位置在最新的位置
