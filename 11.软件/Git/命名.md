# 命名

## 命名规范

+ 这是 Git 分支命名规范，建议命名为

  + feature 用于开发新功能
  + bugfix 用于修复 bug
  + hotfix 用于紧急修复生产环境中的 bug
  + release 用于准备发布新版本
  + chore 对构建过程或辅助工具和库（如文档）的更改
  + docs 仅文档更改
  + refactor 代码重构（既不修复 bug 也不新增功能）
  + perf 改进性能的代码更改
  + test 添加缺少的测试或更正现有测试
  + style 不影响代码含义的更改（空白、格式设置、缺失 分号等）
  + 等

+ Subject

  + Subject（主题）：简洁描述提交的目的，不超过 50 个字符

+ Body 部分

  + 详细描述提交的内容和背景
  + 说明为什么需要这次提交，以及如何实现的
  + 每行不超过 72 个字符
