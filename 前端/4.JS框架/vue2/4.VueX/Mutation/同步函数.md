# 同步函数

## Mutation 必须是同步函数

  - 要记住 mutation 必须是同步函数。why？

    ```js
    mutations: {
      [COUNT_INCREMENT] (state) {
        setTimeout(() => {
          state.count ++;
        }, 1000)
      },
    }
    ```

  - 执行上端代码，我们会发现更改 `state` 的操作是在回调函数中执行的，这样会让我们的代码在 `devtools` 中变的不好调试：当 `mutation` 触发的时候，回调函数还没有被调用，`devtools` 不知道什么时候回调函数实际上被调用，任何在回调函数中进行的状态的改变都是不可追踪的
