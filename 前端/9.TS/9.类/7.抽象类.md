# 抽象类

## 为什么需要抽象类

+ 有时某个类指标是一个抽象概念，主要用于提取子类共有的成员，而不能直接创建它的对象

+ 该类可以作为抽象类

## abstract

+ 给类前面加上 `abstract` ，表示该类是一个抽象类

+ 不可以创建一个抽象类

 ```ts
  // 添加 abstract 关键字
  abstract class Chess {}

  class Horse extends Chess {}

  const h = new Horse();

  const c = new Chess(); // 报错 无法创建抽象类的实例
  ```

## 抽象成员

+ 父类中可能知道有些成员是必须存在的，但不知道该成员的值或实现是什么

+ 因此需要有一种强约束，让继承该类的子类，必须要实现该成员

+ 抽象类中，可以有抽象成员，这些抽象成员必须在子类中实现

+ 抽象成员 **必须** 在抽象类中

 ```ts
  // 错误
  class Chess {
    x:number= 0;
    y:number= 0;

    // 报错 抽象方法只能出现在抽象类中
    abstract readonly name:string;
  }
  ```

 ```ts
  // 正确
  abstract class Chess {
    x:number= 0;
    y:number= 0;

    // 报错 抽象方法只能出现在抽象类中
    abstract readonly name:string;
  }
  ```

## 示例

+ 示例1

 ```ts
  // 抽象类
  abstract class Chess {
    x:number= 0;
    y:number= 0;

    // 抽象属性
    abstract readonly name:string;

    // 抽象方法
    abstract move(targetX:number, targetY:number):boolean
  }

  class Horse extends Chess {
    // 实现抽象属性
    readonly name: string = "马"

    // 实现抽象方法
    move(targetX:number, targetY:number) {
      this.x = targetX;
      this.y = targetY;
      console.log(`${ this.name }移动成功` );
      return true
    }
  }

  class Pao extends Chess {
    // 实现抽象属性
    readonly name: string;

    constructor() {
      super();
      // 实现抽象属性
      this.name = "炮"
    }

    // 实现抽象方法
    move(targetX:number, targetY:number) {
      this.x = targetX;
      this.y = targetY;
      console.log(`${ this.name }移动成功` );
      return true
    }
  }

  class Soldier extends Chess {
    // 实现抽象属性
    get name() {
      return "兵"
    }

    // 实现抽象方法
    move(targetX:number, targetY:number) {
      this.x = targetX;
      this.y = targetY;
      console.log(`${ this.name }移动成功` );
      return true
    }
  }

  const h = new Horse();
  const p = new Pao();
  const s = new Soldier();

  h.move(1, 2); // 马移动成功
  p.move(1, 2); // 炮移动成功
  s.move(1, 2); // 兵移动成功
  ```

+ 示例2：使用 模板模式

 ```ts
  // 抽象类
  abstract class Chess {
    x:number= 0;
    y:number= 0;

    // 抽象属性
    abstract readonly name:string;

    move(targetX:number, targetY:number):boolean {
      // 1.边界判断
      this.isOutSide();

      // 2.目标位置是否有己方棋子
      this.targetHasMengyou();

      // 3.棋子移动规则判断
      if(this.rule(targetX, targetY)) {
        this.x = targetX;
        this.y = targetY;
        console.log(`${ this.name } 移动成功`);
        return true
      }
      return false
    }

    protected isOutSide():boolean {
      console.log("边界判断");
      return true
    }

    protected targetHasMengyou():boolean {
      console.log("目标位置是否有己方棋子");
      return true
    }

    protected abstract rule(targetX:number, targetY:number):boolean
  }

  class Horse extends Chess {
    // 实现抽象属性
    readonly name: string = "马"

    rule(targetX:number, targetY:number):boolean {
      return true
    }
  }

  class Pao extends Chess {
    // 实现抽象属性
    readonly name: string;

    constructor() {
      super();
      // 实现抽象属性
      this.name = "炮"
    }


    rule(targetX:number, targetY:number):boolean {
      return true
    }
  }

  class Soldier extends Chess {
    // 实现抽象属性
    get name() {
      return "兵"
    }


    rule(targetX:number, targetY:number):boolean {
      return true
    }
  }

  const h = new Horse();
  const p = new Pao();
  const s = new Soldier();

  h.move(1, 2); // 马移动成功
  p.move(1, 2); // 炮移动成功
  s.move(1, 2); // 兵移动成功
  ```
