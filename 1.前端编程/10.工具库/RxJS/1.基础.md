## RxJS 基本概念解释

+ Observable (可观测对象): 数据哪里来的，怎么持续产生的
+ Observer (观察者):  值来了怎么接收，错了怎么办，结束了怎么办
+ Subscription (订阅): 什么时候接受，什么时候停止
+ Operators (运算符): 数据流怎么变换、过滤、组合
+ Subject: 一个数据源怎么推给多个订阅者
+ Schedulers (调度器): 什么时候执行，在那种调度方式中执行

+ 综合使用

  + 有一个 `Observable` 在未来不断地产生数据、不断的产出值， `Observer` 可以订阅（ `subscribe` ）它
  + `subscribe` 返回一个 `Subscription` 。返回 `Subscription` 的意义在于后面的控制（例如：取消）
  + 数据在流动的过程中可以经过 `Operators` 做处理
  + 如果需要一份数据同时发给多个人，则可以使用 `Subject` 。如果你还要控制这件事什么时候执行、怎么进行调度，就会用到 `Schedulers`

## 使用示例

+ 示例1

  ```js
  // 输入框搜索
  import { fromEvent, of,from } from "rxjs";
  import { map, filter, debounceTime, distinctUntilChanged, switchMap, catchError } from "rxjs/operators";

  const input = document.querySelector("#search");
  const results = document.querySelector("#results");

  fromEvent(input, "input")
    .pipe(
      map((e) => e.target.value.trim()), // 取值并去除两端空白
      filter((text) => text.length > 0), // 过滤空输入
      debounceTime(500), // 防抖：500ms内没有新输入才继续
      distinctUntilChanged(), // 去重：只有输入变化才继续
      // 版本号+取消前一个请求解决竞态问题
      switchMap((keyword) =>
        from(fetch(`/api/search?q=${encodeURIComponent(keyword)}`).then((res) => res.json())).pipe(
          // 错误兜底
          catchError((err) => {
            console.error("搜索请求失败:", err);
            return of({ items: [] }); // 出错时返回空结果
          })
        )
      ),
    )
    .subscribe((data) => {
      results.innerHTML = data.items.map((item) => `<li>${item}</li>`).join("");
    });
  ``
