# 基础

## 操作符

+ 操作符

  | 操作符                   | 用途                                 | 典型场景                 |
  | ------------------------ | ------------------------------------ | ------------------------ |
  | `filter(fn)`             | 条件过滤                             | 非空检查、权限判断       |
  | `distinctUntilChanged()` | 连续相同值去重                       | 输入去重、状态无变化跳过 |
  | `take(n)`                | 只取前 N 个值然后 complete           | 调试、一次性操作         |
  | `takeUntil(notifier$)`   | 收到信号就停                         | 组件卸载时停止流         |
  | `skip(n)`                | 跳过前 N 个值                        | 忽略初始值               |
  | `first()`                | 取第一个值后 complete                | 等第一次事件             |
  | `last()`                 | 取最后一个值（源 complete 后才发出） | 取最终结果               |
  | `takeWhile(fn)`          | 条件为 true 就继续，false 就停       | 持续监听直到满足条件     |

## 思想

+ 过滤操作符本质上是在做**噪声消除**。真实的数据源（用户输入、WebSocket 消息、传感器数据）充满了冗余和无效信号

+ 过滤操作符在管道的前端就把它们剔除，让后面的昂贵操作只处理有意义的数据

## 对比

+ take 系列的快速对比

  ```js
  // take(3)：取前 3 个值就结束
  rxjs.interval(1000).pipe(rxjs.operators.take(3))
    .subscribe(n => console.log("take:", n));
  // 0, 1, 2，然后 complete

  // first()：等价于 take(1)，但更语义化
  rxjs.interval(1000).pipe(rxjs.operators.first())
    .subscribe(n => console.log("first:", n));
  // 0，然后 complete

  // takeWhile：条件满足就继续，不满足就停
  rxjs.interval(1000).pipe(
    rxjs.operators.takeWhile(n => n < 3)
  ).subscribe(n => console.log("takeWhile:", n));
  // 0, 1, 2，n=3 时条件为 false → complete
  ```
