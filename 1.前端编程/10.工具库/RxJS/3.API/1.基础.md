# 基础

## 概述

+ RxJS 有 100+ 个操作符，但你在日常开发中用到的不超过 20 个，其实它们分布在清晰的 不同类别 里

## 分类

+ 第一类 创建 Creation
+ 第二类 转换 Transformation
+ 第三类 过滤 Filtering
+ 第四类 时间控制 Time-based
+ 第五类 高阶操作符 Higher-Order / Flattening
+ 第六类 组合 Combination
+ 第七类 错误处理与生命周期

## 总结

  ```
  数据从哪来？            → 创建（Creation）
  数据变个形？            → 转换（Transformation）
  哪些数据不要？          → 过滤（Filtering）
  频率怎么控制？          → 时间控制（Time-based）
  新异步来了旧的咋办？    → 高阶 / 展平（Higher-Order）
  多条流怎么协作？        → 组合（Combination）
  出问题了怎么办？        → 错误处理与生命周期（Error & Lifecycle）
  ```

这七个类别对应的是你处理**任何异步数据流时都会遇到的七个问题**。不管 RxJS 的 API 将来怎么变，这七个问题不会变——它们是异步编程的本质维度。

> 当你遇到一个新需求时，不要先想"用哪个操作符"，而是先问"这是哪一类问题"。比如："用户连续点击按钮，之前的请求还没回来"——这是高阶操作符的问题（新旧任务关系），而不是时间控制的问题。"页面初始化时同时发三个请求，全部回来了才渲染"——这是组合操作符的问题（forkJoin）。定位到类别后，那一类里通常只有 2-4 个选项，选择就简单了。
