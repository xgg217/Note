# 基础

## 概述

+ 转换操作符把流中的值从一种形状变成另一种形状

## 操作符

+ 操作符

  | 操作符           | 用途                               | 典型场景                          |
  | ---------------- | ---------------------------------- | --------------------------------- |
  | `map(fn)`        | 每个值经过函数转换                 | 提取字段、格式化                  |
  | `scan(fn, seed)` | 累积计算（像 reduce 但每步都输出） | 累加器、状态聚合                  |
  | `tap(fn)`        | **不改值**，只"看一眼"或执行副作用 | 调试日志、埋点、更新 loading 状态 |

## 已废弃的API

+ 老教程可能提到 `pluck` 和 `mapTo`，它们在 RxJS 7 中已废弃，直接用 `map` 代替即可
+ 比如 `pluck('target', 'value')` 换成 `map(e => e.target.value)`

## tap

+ **为什么 `tap` 放在转换类？**
+ 严格说 `tap` 不做转换——它收到什么值就原样传给下游
+ 但它在管道中的**位置**和转换操作符一样——夹在数据流中间。你可以把它理解成"转换系数为 1 的 map"
+ RxJS 官方把它归为 Utility，但从使用频率和位置来看，和 `map` 放一起更好记
