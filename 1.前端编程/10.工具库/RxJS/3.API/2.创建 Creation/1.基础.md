# 基础

## 概述

+ 创建操作符是流的入口点
+ 你不是凭空写 `new Observable` ，而是用创建操作符把各种数据源（DOM 事件、数组、Promise、定时器）包装成 Observable

## 设计意图

+ 创建操作符解决的是"桥接"问题——把非 Observable 的世界（DOM、Promise、原始值）桥接到 Observable 的世界
+ 之后所有处理都在管道内完成

## 操作符

+ 操作符

  | 操作符                 | 用途                         | 典型场景             |
  | ---------------------- | ---------------------------- | -------------------- |
  | `fromEvent(el, type)`  | DOM 事件 → 流                | 点击、输入、滚动     |
  | `of(a, b, c)`          | 给定值 → 同步流              | 测试、兜底值         |
  | `from(arrayOrPromise)` | 数组/Promise/可迭代对象 → 流 | 将已有数据接入管道   |
  | `interval(ms)`         | 定时器 → 流（0, 1, 2, ...）  | 轮询、心跳           |
  | `timer(delay, inter)`  | 延迟 + 可选间隔 → 流         | 一次性延时或定时重复 |
  | `defer(() => ...)`     | 延迟创建，subscribe 时才决定 | 每次订阅新建 Promise |
  | `EMPTY`                | 空流，立刻 complete          | catchError 静默兜底  |
  | `throwError(() => e)`  | 直接抛错的流                 | 测试 error 路径      |

## of vs from 区别

+ of vs from 常问的区别

  ```js
  of([1, 2, 3]).subscribe(v => console.log(v));
  // 打印一次：[1, 2, 3]（一个值：整个数组）
  ```

  ```js
  from([1, 2, 3]).subscribe(v => console.log(v));
  // 打印三次：1  2  3（三个值：逐个元素）
  ```

## from(promise) vs defer(() => promise) 区别

+ from(promise) vs defer(() => promise) 的关键区别

  ```js
  // from：Promise 在定义时就已经开始执行了
  const p = fetch("/api/user"); // 请求已经发出
  const a$ = from(p);           // 只是包装，不影响执行时机

  // defer：每次 subscribe 才开始执行
  const b$ = defer(() => fetch("/api/user")); // 此时没有请求
  b$.subscribe(); // 这时候才发请求
  b$.subscribe(); // 再发一次——独立的
  ```

## interval 和 timer 对比

+ interval 和 timer 的快速对比

  ```js
  // interval：从 0 开始，每隔 1 秒发一个递增数字，永不停止
  rxjs.interval(1000).pipe(
    rxjs.operators.take(5) // 只取前 5 个，否则永远不停
  ).subscribe(n => console.log("interval:", n));
  // 输出：0, 1, 2, 3, 4（每秒一个）

  // timer(3000)：等 3 秒后发一个 0，然后 complete
  rxjs.timer(3000).subscribe(n => console.log("timer:", n));
  // 3 秒后输出：0

  // timer(1000, 2000)：等 1 秒开始，然后每 2 秒发一个
  rxjs.timer(1000, 2000).pipe(
    rxjs.operators.take(3)
  ).subscribe(n => console.log("timer+interval:", n));
  // 1 秒后：0，3 秒后：1，5 秒后：2
  ```
