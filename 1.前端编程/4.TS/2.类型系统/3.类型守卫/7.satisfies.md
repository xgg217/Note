# satisfies

## 概述

+ `satisfies` 是一个类型操作符，它是TS4.9的新功能

## 解决的问题

+ 它解决了一个微妙但令人沮丧的问题：在不丢失对象具体细节的情况下，对对象的结构进行校验

+ 想象一位老师在检查你的作业。

  + 他们不会重写你的答案。他们只是检查：

    + “是的，这符合规则。”

+ 这正是 satisfies 所做的事情

+ 它会检查你的对象是否符合所需的结构，但不会改变你最初写下的内容

+ 因此你获得：

  + 校验

  + 安全性

  + 精确性

## 普通类型注解的问题

+ 当你这样写时：

  ```ts
  type ButtonVariant = "primary" | "secondary";
  const button: ButtonVariant = "primary";
  ```

+ TypeScript 只会记住：

  + 这是允许的变体之一

+ 但如果你使用对象：

  ```ts
  type ButtonConfig = {
    variant: "primary" | "secondary";
  };
  const config: ButtonConfig = {
    variant: "primary"
  }
  ```

+ TypeScript 会发生类型扩宽

  + 它关注的是匹配类型，而不是保留精确的字面量细节

## 更好的方式 —— satisfies

+ satisfies

  ```js
  type ButtonConfig = {
    variant: "primary" | "secondary";
  };

  const config = {
    variant: "primary"
  } satisfies ButtonConfig;
  ```

+ 现在会发生两件事：

  + TypeScript 会检查该对象是否符合 ButtonConfig

  + 它仍然会记住精确的值 "primary"

## 使用场景2 防止拼写错误

+ satisfies 可以捕获那些简单类型注解可能无法清晰提示的错误

  ```ts
  type ButtonConfig = {
    variant: "primary" | "secondary";
  };

  const config = {
    varient: "primary" // ❌ 拼写错误
  } satisfies ButtonConfig;
  ```

## 使用场景 3 配置

+ 当处理配置对象、路由或映射时

  ```js
  type Routes = Record<string, string>;
  const routes = {
    home: "/",
    about: "/about",
    contact: "/contact"
  } satisfies Routes;
  ```

+ 现在 TypeScript：

  + 确保所有值都是 string

  + 保留精确的键名

  + 保留字面量类型
