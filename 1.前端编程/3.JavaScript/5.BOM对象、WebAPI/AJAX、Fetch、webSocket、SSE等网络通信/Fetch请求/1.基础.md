# 基础

## XMLHttpRequest 问题

1. 所有的功能全部集中在一个对象上，容易书写混乱，不易维护代码

2. 采用传统的事件驱动模式，无法适配新的 `Promise Api`

## Fetch Api 特点

+ 并非取代 `AJAX` ，而是对 `AJAX` 传统 `API` 的改进

+ 细分的功能分割：头部信息，请求信息、响应信息等均分布到不同的对象，更利用处理各种复杂的 `AJAX` 场景

+ 使用 `Promise Api` ，更利于异步代码的书写

+ `Fetch Api` 并非 ES6 的知识，属于 HTML5 新增的 `Web Api`

+ 需要掌握网络通信的知识
