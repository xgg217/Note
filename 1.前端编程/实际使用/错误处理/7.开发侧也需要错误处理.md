# 开发侧也需要错误处理

## 概述

+ 错误处理不只是给用户看的，也是给开发排查问题用的。

+ 很多线上问题最痛苦的地方不是错误本身，而是你不知道发生了什么。

+ 用户只说：

  + 页面点不了。

+ 或者：

  + 提交失败了。

+ 但开发需要知道：

  + 用户在哪个页面？

  + 点了哪个按钮？

  + 请求发出去了吗？

  + 接口返回了什么？

  + 当前用户是什么角色？

  + 浏览器是什么版本？

  + 是否有前端运行时错误？

  + 这次请求的 requestId 是什么？

+ 如果没有这些信息，排查问题就只能靠猜。

+ 所以成熟项目里，前端错误处理通常还要配合监控和日志。

+ 比如：

  + 捕获运行时错误

  + 捕获未处理的 Promise rejection

  + 记录接口失败

  + 记录关键用户操作

  + 上报页面白屏

  + 保留 source map 用于定位源码

  + 把 requestId 展示或记录下来，方便和后端日志关联

+ 这些东西不一定一开始就全部做完

+ 但至少要有意识：错误处理不仅要让用户知道怎么办，也要让开发知道去哪查
