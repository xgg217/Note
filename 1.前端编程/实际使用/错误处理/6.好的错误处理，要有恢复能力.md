# 好的错误处理，要有恢复能力

## 概述

+ 错误处理最容易被忽略的一点是恢复能力。

+ 很多代码只是告诉用户失败了，但没有帮助用户恢复。

  + 比如表单提交失败后，表单被清空了。这是非常糟糕的体验。因为用户真正损失的不是一次请求，而是刚才输入的内容。

  + 再比如上传文件失败后，文件选择状态没了，用户还要重新选一次文件。

  + 又比如 AI 生成失败后，原输入被覆盖了，用户无法回到之前的内容。

+ 这些都不是简单的“失败提示”能解决的。

+ 好的错误处理应该尽量做到：

  + 保留用户输入

  + 保留用户选择

  + 允许重试

  + 允许撤销

  + 允许回到失败前状态

  + 明确告诉用户下一步动作

+ 比如保存文章失败时，可以提示：

  ```
  保存失败，当前内容已保留，你可以稍后重试。
  ```

+ 这句话比单纯的“保存失败”有用得多。

+ 因为它告诉用户两件事：

  1. 保存没有成功

  2. 内容没有丢

+ 用户知道内容没丢，焦虑感就会降低很多。

+ 所以错误处理的目标不是展示错误

+ 更准确地说：错误处理的目标，是帮助用户从错误中恢复
