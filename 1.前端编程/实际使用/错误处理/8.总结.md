# 总结

## 概述

+ 很多前端能力差距，不体现在正常流程里。因为正常流程大家都能写。点击按钮，请求接口，更新页面，提示成功，这些并不难。

+ 真正拉开差距的，是异常路径。成熟一点的前端，会继续追问：

  + 失败了怎么办？

  + 慢了怎么办？

  + 重复提交怎么办？

  + 用户离开了怎么办？

  + 数据丢了怎么办？

  + 权限变了怎么办？

  + 怎么恢复？

  + 怎么排查？

+ 这些问题看起来都在主流程之外。但它们决定了一个功能是不是真的可靠。

+ 所以，错误处理不是写完功能之后顺手补一个 catch。它应该从一开始就参与功能设计。

+ 如果用一句话总结，我觉得应该是：成熟的前端代码，不是只把成功路径写通，而是能把失败、取消、超时、重试、兜底和恢复都设计清楚。

+ 这也是为什么我说：错误处理，是前端最容易忽略，但也最能体现工程能力的地方
